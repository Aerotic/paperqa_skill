简述

本文提出 MobileGPT，一个通过模拟人类式应用记忆来增强大语言模型（LLMs）的移动任务自动化系统。核心问题在于使 LLM 能够跨不同应用页面高效学习并复用子任务，因为现有数据集未能捕捉重复性任务。MobileGPT 采用三层层次化记忆架构，将任务自动化分解为探索（Explore）、选择（Select）、推导（Derive）与回忆（Recall）四个阶段，结合屏幕分类与人在回路修正，在温启动（warm-start）下取得 82.5% 任务成功率，相较基线方法 AutoDroid 和 AppAgent 大幅降低延迟达 87%，且离线探索以 $10.78 成本覆盖所需页面 89.65%。

📋

问题与挑战

移动任务自动化中，LLM 需要处理不同应用页面上的重复性子任务，但现有数据集与系统缺乏对子任务的结构化分解与跨页面记忆复用能力，导致效率低、延迟高。传统方法直接从 UI 元素列表中推导操作，未能识别相似功能页面并复用已学策略，且易出现屏幕分类错误（如大量假阳性/假阴性），导致任务执行失败或需反复从头推理。

📋

方法概览

MobileGPT 设计了一套以层次化记忆为核心的自动化工作流。它将应用页面按功能语义归类为节点，构建三层记忆（任务-子任务-动作）。系统采用冷启动（cold-start）阶段的探索-选择-推导流程来学习新任务，利用 GPT-4-turbo 驱动；温启动阶段则通过子任务槽填充和上下文动作适应来回忆并复用已有记忆，使用 GPT-3.5-turbo 执行。屏幕分类通过验证关键 UI 元素而非提取完整子任务列表来实现，并引入双策略修正机制，将 LLM 自纠正与人在回路修复结合以保障可靠性。

探索-选择-推导-回忆工作流与层次化记忆

  • 核心思想：将任务自动化分解为四个阶段，并构建三层层次化记忆（任务-子任务-动作），使系统能将页面按功能语义聚类到节点，实现子任务级别的结构化解耦与复用。
  • 与已有工作的区别：以往系统如 AutoDroid 缺乏子任务分解和记忆复用机制，每次需从完整 UI 列表从头推导，导致重复计算和高延迟。MobileGPT 首次引入分层记忆与阶段式学习-回忆流程。
  • 技术实现：冷启动阶段使用 GPT-4-turbo 依次完成探索、选择、推导，温启动阶段通过 GPT-3.5-turbo 进行槽填充和动作适应，复用先前存储的记忆节点与子任务序列。
  • 有效性证据：各阶段准确率达探索 96.4%、选择 96.2%、推导 99.1%，步骤准确率 99.8%。温启动相比 AutoDroid 延迟降低 87%，证实工作流与记忆架构显著提升效率与精度。
💡

基于子任务的屏幕分类

  • 核心思想：不再提取完整子任务列表，而是通过比对当前屏幕与记忆中关键 UI 元素集的余弦相似度，快速判定屏幕是否属于已知节点，实现轻量级的高效分类。
  • 与已有工作的区别：传统方法依赖完整 UI 元素列表直接分类，在测试中报告了 38 个假阳性和 57 个假阴性；MobileGPT 的方法极大抑制了误判，使记忆召回更可靠。
  • 技术实现：记忆存储每个节点的一组关键 UI 元素特征向量，运行时实时计算相似度，以阈值判定匹配。
  • 有效性证据：在 269 个应用屏幕上，仅出现 3 个假阳性和 0 个假阴性，假阳性减少 92.1%，假阴性完全消除。

双策略修正机制

  • 核心思想：结合 LLM 自我纠正与人在回路（Human-in-the-loop, HITL）修复，规则检测 UI 错误和循环后先由 LLM 尝试修复，失败则提升至人工干预，以此兼顾自动化与鲁棒性。
  • 与已有工作的区别：多数自动化代理仅依赖单次推理或简单重试，缺乏分层容错设计；MobileGPT 的累积式修正将人在回路精确介入在探索、选择、推导阶段，最大化修复效果。
  • 技术实现：基于规则的反馈模块监测执行结果，触发 LLM 基于错误信息生成修正动作；若仍失败，系统在对应阶段请求人工提供正确步骤并存入记忆。
  • 有效性证据：温启动准确率从无人在回路时的 72.3% 跃升至含人在回路时的 98.75%，提升幅度达 36.6%，证明双策略机制对可靠性的关键作用。

离线随机探索与页面收集

  • 核心思想：在任务执行前，利用随机探索器和用户轨迹监控器自动遍历应用，收集 50 个唯一页面以构建记忆节点，以极小的一次性成本获得高覆盖率的页面预采。
  • 与已有工作的区别：先前方法通常依赖在线探索或人工预定义页面，缺乏系统化的离线自动收集策略，导致覆盖不全或上线后延迟高。
  • 技术实现：随机探索器按照启发式策略点击可交互元素，结合用户轨迹监控避免重复和无意义页面，将采集的页面经聚类形成记忆节点。
  • 有效性证据：收集的页面覆盖了所需测试页面的 89.65%，总成本仅为 $10.78，显示出极高的性价比和实用性。

基于属性与上下文的动作适应

  • 核心思想：在温启动回忆子任务模板后，通过槽填充绑定具体实体值，并利用 LLM 少样本上下文学习动态调整动作参数，以应对 UI 元素属性变化。
  • 与已有工作的区别：传统方法或固定动作序列，无法适配参数变化；MobileGPT 将参数绑定与动作微调解耦，实现高泛化能力的动作复用。
  • 技术实现：槽填充由 GPT-3.5-turbo 根据屏幕内容填充模板中的槽位；上下文适应则提供少量示例，使 LLM 能够修改坐标、文本等动作细节。
  • 有效性证据：槽填充准确率达 99.5%，上下文适应准确率 100%，整体步骤准确率 99.8%，表明该机制几乎完美地解决了动作参数变化的适应问题。
