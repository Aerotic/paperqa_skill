Based solely on the provided context, the paper's major innovations are as follows:

**1. Three-Phase Inference Structure (Explore-Select-Derive)**
The core technical idea decomposes mobile task completion into distinct phases: Explore (proactively collects sub-tasks per app screen), Select (LLM picks the most pertinent sub-task, including global ones like 'Read_Screen' and 'Finish'), and Derive (LLM incrementally selects low-level actions like click/input/scroll) (lee2024mobilegptaugmentingllm pages 5-6). This differs from a traditional "Derive-only" baseline structure (lee2024mobilegptaugmentingllm pages 10-11). Implementation uses multiple LLM agents: GPT-4-turbo for screen understanding and reasoning, GPT-3.5-turbo for slot-filling (lee2024mobilegptaugmentingllm pages 8-9). Quantitative evidence from the ablation study shows phase accuracy: Explore at 96.4%, Select at 96.2%, and Derive at 99.1% (lee2024mobilegptaugmentingllm pages 10-11).

**2. Human-Like App Memory with Three-Level Hierarchy**
The system introduces a three-level hierarchical memory (tasks, sub-tasks, actions) for efficient storage and reuse (lee2024mobilegptaugmentingllm pages 3-4). This enables shared learning across tasks and addresses challenges in efficient storage and flexible recall (lee2024mobilegptaugmentingllm pages 10-11, lee2024mobilegptaugmentingllm pages 3-4). 

**3. Dual-Strategy Failure Handling**
The paper implements a dual-strategy correction mechanism: (1) Self-correcting via rule-based feedback (e.g., 'There is no UI with index i', 'You have looped the same screens X times') appended to the prompt; (2) Human-in-the-loop (HITL) task repair for Explore/Select/Derive failures (lee2024mobilegptaugmentingllm pages 5-6). The repair mechanism scored 64.6 on the System Usability Scale (SUS) (lee2024mobilegptaugmentingllm pages 12-13).

**4. Offline Exploration for App Page Discovery**
Offline exploration uses a random explorer and user trace monitor (lee2024mobilegptaugmentingllm pages 3-4). It discovered 50 app pages per app (89.65% of needed pages) at a monetary cost of $10.78 (lee2024mobilegptaugmentingllm pages 10-11).

**5. Function-Call Parameterization**
The system uses function-call parameterization for handling task variations (lee2024mobilegptaugmentingllm pages 3-4). This is described as an innovation for flexible task recall.

**6. App Launch Recommendation via Vector Database**
App launch recommends a top-3 list using Google Play descriptions stored in a vector database (lee2024mobilegptaugmentingllm pages 8-9).
